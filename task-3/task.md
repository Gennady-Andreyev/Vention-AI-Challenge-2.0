# Task

In this task you will use n8n to build an AI-powered personal learning assistant delivered as a Telegram bot. The system helps users learn new material through intelligent summarization and quiz generation. It processes learning materials submitted as URLs, creates structured summaries, generates comprehension quizzes.

---

**No extra subscriptions needed for this one — a free n8n trial is enough, and n8n includes free GPT tokens for AI nodes out of the box.**

---

## AI roles

Your workflow must implement two distinct AI roles:

- **Teacher** — analyzes submitted learning materials, creates structured summaries with key concepts, identifies difficulty level, and formats content for easy understanding.
- **Examiner** — generates multiple choice quiz questions, creates correct answers, validates user responses, and provides explanations for incorrect answers.

## How the system works

### Step 1 — Bot setup and commands

- Users interact with the bot via a Telegram interface.
- The bot supports three commands: /start, /learn [url], and /quiz.

### Step 2 — Material processing

- The user sends /learn [URL]; an HTTP Request node fetches and extracts the content.
- The Teacher AI agent analyzes the content and creates a structured summary: five to seven key points, main concepts, and difficulty level.
- The cleaned content, summary, and title are saved for future quiz generation.
- The bot sends the formatted summary back to the user.

### Step 3 — Quiz generation

- When the user requests a quiz — either via /quiz or inline after learning — the Examiner AI agent generates five questions.
- If triggered via /quiz, the user first selects a topic from their saved materials.
- Questions are sent one by one with inline keyboard buttons for answer selection.
- User responses are captured and stored.

### Step 4 — Answer validation and feedback

- The Examiner AI agent validates answers and calculates a score as a percentage.
- For incorrect answers, the agent generates explanations.
- The bot sends a results summary with per-question feedback.

## Requirements

### Application behavior

- The Telegram bot responds correctly to all three commands: /start, /learn [url], and /quiz.
- Material processing extracts real content from submitted URLs — summaries must be relevant to the specific URL, not generic.
- The Teacher agent produces a structured summary with five to seven key points, main concepts, and a difficulty level.
- The Examiner agent generates five quiz questions specific to the submitted material — questions must not be hardcoded or reused across different materials.
- Answer validation is intelligent — it does not rely on exact text match only.
- Data persists between sessions — users can return to previously saved materials and their progress is retained.
- When triggered via /quiz, the user is presented with a list of their saved topics to choose from.
- The bot does not require a workflow restart between commands.
